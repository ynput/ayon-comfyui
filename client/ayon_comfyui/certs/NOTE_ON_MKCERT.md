Notes on mkcert
---

Please [download mkcert](https://github.com/FiloSottile/mkcert/releases) from the official repository (as linked) and run one of the shell scripts after ensuring mkcert is in PATH environment. 

[mkcert](https://github.com/FiloSottile/mkcert) is a cross platform tool that allows you to update the certificate root store, trusting your own PC as a certificate root authority. This allows you to then generate certificates to trust whichever adress you want, but most importantly, your own. Otherwise, you would run into mixed security errors in browsers, for example.

In particular, we use this to make our plugin-hosted server available to comfyui instances that are served through https, instead of http.

This is roughly the commands that https_helper.py uses.

```sh
mkcert -install
mkcert -cert-file comfyui-cert.pem -key-file comfyui-key.pem localhost 127.0.0.1
```

The CA generated by mkcert is **unique to every machine**. There is no distributing going on.
A distributed CA could meaningfully impact the security of a network.

The certificates are re-generated every session.
